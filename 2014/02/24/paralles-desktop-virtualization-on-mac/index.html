Parallels Desktop - Virtualization on Mac

2014-02-24
works

Parallels Desktop支持在 Windows 与 Mac 应用程序之间拖放文件并直接从 Mac dock 启动 Windows 应用程序。

由于工作的原因, 我需要在Mac下安装Windows系统. 尽管这是一件比较二的事情但却不得不为之。

经过一番的调查研究，在Mac下安装双系统有两种方式： Boot Camp或者虚拟机。Apple提供的Boot Camp能够让用户在Mac上以原生的方式运行Windows操作系统，然而切换系统需要重启机器，这对于需要同时工作在双系统下的我来说很不方便，于是我选择了在Mac上安装虚拟机。

目前市场上比较好的虚拟机软件当属Vmware Fusion和Parallels Desktop了。在纵览了一堆评测文章后发现两者并没有什么区别，我试用了两者一段时间后最终选择了Parallels Desktop，主要看中了它支持Mac与Windows系统集成的强大功能，比如在Mac下可以使用Windows应用，共享目录等，另外Parallels Desktop的使用界面、配置安装等相对较为方便，适合我的品味。

Parallels Desktop For Mac

Parallels Desktop 虚拟机安装主要有三步：选定虚拟机镜像，进行配置、安装，很简单吧。我在安装过程中发现对于配置与Mac的集成方式这部分比较有用，所以就在本文分享一下。如果想了解具体的安装可以看一下官网的视频。

虚拟机与Mac的集成方式

这里的集成指Mac系统与Windows系统资源与操作一起使用的常见问题。

设置集成方式

Parallels Desktop提供了两种方式：
{:height=”300px” width=”400px”}

  • 1 Like Mac

这种模式下，Mac系统与Windows系统将共享用户Profile，即桌面和User Home文件夹在两个系统中会是同一个。而Windows系统下的程序会作为一个Mac的应用出现在Dock上或者Notification Bar，读者可以通过改变这些参数来体验不同的行为。

如下图所示：
{:height=”300px” width=”400px”}

{:height=”300px” width=”400px”}

  • 2 Like PC

即所有Windows系统的程序将出现在同一个窗口中，就像平常用的虚拟机的工作方式，将Windows系统与Mac相隔离，但这并不阻止两个系统之间共享文件和复制剪切文本。

更改集成方式

Parallels Desktop提供了配置选项可以用来动态调整集成方式.

通过Parallels Desktop 菜单 => Virtual Machine => Configure 打开配置选项，其中Sharing、Applications、Advanced、Coherence几个选项决定了集成方式的不同。

{:height=”300px” width=”400px”}

  • Options => Sharing

如果启用Shared Profile选项则表示Windows系统与Mac系统会共享用户Home Profile，包括桌面等内容。改变这个选项Prallels Desktop会马上进行切换。

{:height=”300px” width=”400px”}

Share Folders选项则可以配置双系统之间共享那些文件夹，读者可以根据自己的需要进行设置，设置完后会在Windows系统的文件夹中体现。
比如下图是我的文件夹：
{:height=”300px” width=”400px”}

  • Options => Applications

如果启用Share Windows applications with Mac，那么Windows的程序就会加到Mac的Dock中，作为一个单独的Mac应用而出现。

{:height=”300px” width=”400px”}

  • Options -> Advanced

可以配置共享剪切板，支持文本文件的复制剪切功能。
{:height=”300px” width=”400px”}

安装Prallels Desktop Tools

要实现以上集成功能的一个前提就是需要安装Parallels Desktop Tools，这个工具包在Parallels Desktop的Mac应用包中可以找到，安装完毕Windows系统就可以和Mac一起共享资源了，你也来试试吧。
